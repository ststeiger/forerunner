Namespace: reportParameter

$.forerunner. reportParameter

Widget used to manage report parameters
Properties:
Name Type Description
options Object The options for report parameter
Properties
Name Type Description
$reportViewer Object The report viewer widget
$appContainer Object Report page container
pageNum Integer Report page number

Example

$paramArea.reportParameter({ $reportViewer: this });
$("#paramArea").reportParameter({
 $reportViewer: $viewer,
 $appContainer: $appContainer
});

Methods

destroy()

Removes the report parameter functionality completely. This will return the element back to its pre-init state.

formIsValid() → {Boolean}

Returns a boolean that indicates if the form fields are all valid. Note a form with no fields is considered valid.
Returns:
- true if the all fields of the from are valid
Type
Boolean

getNumOfVisibleParameters() → {Integer}

Get the number of visible parameters
Returns:
The number of visible parameters.
Type
Integer

getParamsList(noValid) → {String}

Generate parameter value list into string and return
Parameters:
Name Type Description
noValid Boolean If not need valid form set noValid = true
Returns:
- Parameter value list or null if this report has no visible parameters
Type
String

removeParameter()

Remove all parameters from report

revertParameters()

Revert any unsubmitted parameters, called in two scenario: 1. when cancelling out from parameter area or before submitting an action 2. when the set of parameters for the session does not match the loaded report.
